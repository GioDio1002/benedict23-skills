维度定义与数据采集

本页给出对比页与雷达图所用的十个维度的精确定义、计算公式与它所表达的「比赛价值」，以及全部数据的获取流程，方便复现与校验。

所有「越低越好」的原始指标在发布时都已经折算成正向对立面（护球率 / 防守压制 / 犯规节制），所以雷达图上「越往外越好」恒成立。
十个维度的定义与价值
维度英文 / 缩写公式它表达了什么
真实命中率TS%PTS / (2 × (FGA + 0.44 × FTA))综合两分、三分、罚球后的得分效率，比单看命中率更能反映「真正能换分的出手」。
使用率USG%100 × (FGA + 0.44×FTA + TOV) × (TmMin/5) / (Min × (TmFGA + 0.44×TmFTA + TmTOV))球员在场时占用回合的比例，衡量出手与失误的责任体量。
助攻率AST%100 × AST / ((Min/(TmMin/5)) × TmFGM − FGM)队友进球中由该球员直接助攻的比例，反映组织端贡献。
队内得分占比Score SharePTS / TmPTS（场均）个人得分占本队得分的份额，刻画球队对其得分的依赖度。
篮板率REB%100 × (REB × (TmMin/5)) / (Min × (TmREB + OppREB))在场时拿到可争抢篮板的比例，去掉了出场时间噪音。
护球率Ball Security100 − TOV%（TOV%=100×TOV/(FGA+0.44×FTA+TOV)）每个使用回合不丢球的比例，越高代表回合终结越干净。
正负值+/-球员在场时本队净胜分（场均）直接反映在场净效果，结合角色读最稳。
防守压制Def. Stops120 − DefRtg（DefRtg=每 100 回合失分）以 120 分为基准减去百回合失分，越高代表在场防守越压制对手。
犯规节制Foul Discipline6 − 场均犯规以 6 犯为基准减去场均犯规，越高代表越少送对方上罚球线。
PIEPIE官方综合贡献：球员各项正向数据 ÷ 比赛中所有球员该项数据之和NBA 官方给出的「整体影响力」单值，便于跨位置横向看。
数据采集流程

1 · 数据来源

以官方 `stats.nba.com` 数据为唯一基准，通过 `nba_api` 调用 `LeagueGameFinder`、`TeamGameLogs`、`BoxScoreTraditionalV3`、`BoxScoreAdvancedV3` 端点，覆盖 2026 总决赛全部 6 场以及两队完整季后赛历程。

2 · 采集策略

按球队 ID + 赛季 ID + SeasonType=Playoffs 拉取场次清单，再按场次 ID 拉每场 box score（基础 + 高阶），原始 JSON 落盘到 `data/raw//` 缓存。

3 · 礼貌抓取

`stats.nba.com` 有显式限速：每次请求间隔 0.6–1.2 秒随机抖动 + 失败指数退避，UA 写明项目身份。再跑分析时只读缓存，绝不二次请求。

4 · 三条基线

每名球员同时构建三组数据：① 本届总决赛逐场聚合；② 该球员完整季后赛均值（含总决赛）；③ 本队总决赛全员均值。三条线在雷达图上同时画出，构成「本人 / 自身基准 / 球队基准」的三角对照。

5 · 衍生指标

派生指标（队内得分占比、护球率 = 100−TOV%、防守压制 = 120−DefRtg、犯规节制 = 6−PF/G）在归一化前显式计算，公式写进 `derive.py` 留底，便于事后复查。

6 · 校验

逐场再用 `BoxScoreSummaryV2` 拉队伍总分 + 胜负，与聚合结果交叉核对；任何球员场均 TS% / USG% 与 `stats.nba.com` 公开页同口径差异超过 0.5 个百分点即报错回查。

7 · 归一化

雷达绘制时按当前页面全集做 per-axis min/max 归一，保证任意两名球员可直接面积对比；表格仍给出原始数值。
